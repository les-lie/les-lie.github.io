How to Pair Program

Vanessa Biggers and Leslie Bell

What is Pair Programming

Pair programming is when two programmers collaborate on coding from a single machine. Usually with two keyboards and two monitors. One programmer is the driver. The driver types the code. The other programmer is the navigator. The navigator tells the driver what to code.



Benefits of Pair Programming

  • Errors are more easily caught with two programmers reviewing the code
  • Programmers who work in pairs learn from each other
  • Pair programming builds team relationships
  • Pair programming keeps programmers focused and motivated


Steps to Successful Pair Programming

  1. Checking in
  2. Identification of roles
  3. Discuss and agree on the scope of the project
  4. Set up testing environment to ensure code satisfaction
