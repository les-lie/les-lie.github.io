Week 2 Technical Reflections

Positioning elements on your page is essential to good design. In addition, you want your code to be easily read by other developers, so it is important to make sure your elements are positioned correctly to facilitate editing and collaboration.

STATIC

In this example, the element we are using is an image. If you place a link to the image inside your CSS without giving it any explicit directions on where to go and how to get there, it will randomly lie somewhere on the page and possibly obstruct anything that is underneath it. We call this STATIC positioning, and it order to achieve it, do nothing. This is the default position of any element defined on your CSS page.


RELATIVE

Using a value of RELATIVE when positioning your element means that the element will be placed exactly the same as STATIC unless you specify, top, right, bottom and/or left parameters. See image shifted right.


FIXED

A FIXED element is positioned on the page relative to the viewable area- that is, when a FIXED element is defined with top, right, bottom, and/or left parameters, it stays put. It always stays in the same place even if the page is scrolled. See upper right corner.


ABSOLUTE

Positioning the element with a value of ABSOLUTE assigns the element a position releative the the previous element that has a position defined in CSS. Positioning your image (element) with a value of ABSOLUTE, you are indeed giving CSS instructions on where you would like the image placed on the page. You are giving instruction that the image is to be placed on the page relative to the last element that received explicit CSS instruction. If there is not previous element to use as a reference point, the value ABSOLUTE positions the element relative to the document BODY. See bottom of page corner.
